
## Исполняемые скрипты проекта

#### make_own_db.py
**Что делает:** Создает локальную базу данных фильмов, загружая информацию с TMDB API.
**Вводные данные:** Требуется действительный API ключ от TMDB (запрашивается при запуске).

#### hello_api_TMDB.py
**Что делает:** Тестирует подключение к TMDB API, проверяет корректность API ключа и делает пробный запрос.
**Вводные данные:** Требуется действительный API ключ от TMDB (запрашивается при запуске).

#### search_in_db.py
**Что делает:** Осуществляет текстовый поиск фильмов по названию в локальной базе данных.
**Вводные данные:** Путь к файлу базы данных (MyFilmDB.json) и ключевое слово для поиска (запрашиваются при запуске).

#### find_similar.py
**Что делает:** Находит похожие фильмы на основе сравнения различных параметров (коллекция, язык, бюджет, жанры).
**Вводные данные:** Путь к файлу базы данных и название фильма для поиска аналогов (запрашиваются при запуске).

### Порядок запуска
1. **make_own_db.py** - первый запуск для создания базы данных
2. **hello_api_TMDB.py** - можно запустить в любое время для проверки API
3. **search_in_db.py** - после создания базы данных для поиска фильмов
4. **find_similar.py** - после создания базы данных для получения рекомендаций

### Требуемые файлы
- Для работы search_in_db.py и find_similar.py необходим файл MyFilmDB.json, созданный make_own_db.py
- Все скрипты требуют установленного Python 3.x

Отлично, давайте разберем первый скрипт `hello_api_TMDB.py` и составим для него документацию.

### Анализ скрипта `hello_api_TMDB.py`

**Что делает скрипт?**
Этот скрипт является простым тестом для проверки работоспособности вашего API-ключа от The Movie Database (TMDB). Он выполняет следующие действия:
1.  Запрашивает у пользователя API-ключ (v3) для доступа к TMDB API.
2.  Проверяет валидность этого ключа, делая пробный запрос к API (`/movie/2`).
3.  Если ключ valid, скрипт делает еще один запрос к API, чтобы получить информацию о бюджете фильма с определенным ID (в данном случае, ID 215).
4.  Выводит значение бюджета этого фильма на экран.

**Сайт, к которому нужен ключ:**
**The Movie Database (TMDB)** - https://www.themoviedb.org/

### Документация для скрипта `hello_api_TMDB.py`

## Скрипт `hello_api_TMDB.py`

## Назначение
Этот скрипт предназначен для проверки корректности вашего API-ключа от сервиса **The Movie Database (TMDB)**. Он выполняет тестовый запрос к API TMDB и выводит бюджет конкретного фильма, демонстрируя, что ключ работает правильно и вы можете получать данные.

## Как запустить?
1.  Убедитесь, что у вас установлен Python 3.
2.  Сохраните скрипт `hello_api_TMDB.py` и вспомогательный модуль `tmdb_helpers.py` в одну папку.
3.  Откройте терминал (командную строку) в этой папке.
4.  Выполните команду:
    ```bash
    python hello_api_TMDB.py
    ```

## Требования для запуска
Для работы скрипта необходим **API-ключ (v3)** от сервиса TMDB.

## Где взять API-ключ?
1.  Перейдите на сайт [The Movie Database (TMDB)](https://www.themoviedb.org/).
2.  Зарегистрируйте новый аккаунт или войдите в существующий.
3.  Перейдите в настройки вашего профиля (Settings).
4.  Выберите вкладку "API" в левом меню.
5.  Запросите API-ключ разработчика (API Key). Обычно нужно заполнить короткую заявку (укажите, что это для обучения/персонального использования).
6.  Скопируйте выданный вам ключ (v3 auth).

## Что он выведет?
При запуске скрипт запросит API-ключ. Ввод ключа скрыт символами (для безопасности).
```
Enter your api key v3:
```
После ввода корректного ключа скрипт выведет бюджет фильма с ID 215 (фильм "Криминальное чтиво"/"Pulp Fiction"):
```
8000000
```
*(Иллюстрация вывода в терминале)*
```bash
user@computer:~$ python hello_api_TMDB.py
Enter your api key v3:
8000000
```

Если ключ неверный, скрипт сообщит об ошибке и завершит работу:
```
Invalid api key
```

## Скрипт `search_in_db.py`

## Назначение
Этот скрипт выполняет поиск фильмов в локальной базе данных (JSON-файле) по ключевому слову. Он ищет совпадения в названиях фильмов (поле `original_title`) и выводит отсортированный список всех найденных фильмов. Это полезно для быстрого просмотра, какие фильмы есть в вашей базе данных.

## Как запустить?
1.  Убедитесь, что у вас установлен Python 3.
2.  Сохраните скрипт `search_in_db.py` и вспомогательный модуль `own_db_helpers.py` в одну папку.
3.  **Предварительно убедитесь, что у вас есть файл базы данных** (например, `MyFilmDB.json`), созданный скриптом `make_own_db.py`.
4.  Откройте терминал (командную строку) в этой папке.
5.  Выполните команду:
    ```bash
    python search_in_db.py
    ```

## Требования для запуска
Для работы скрипта необходим **файл базы данных** в формате JSON, созданный ранее. По умолчанию скрипт ожидает найти файл с именем `MyFilmDB.json` в той же папке, либо вы можете указать полный путь к другому файлу при запросе.

## Где взять необходимую базу данных?
Базу данных необходимо создать с помощью скрипта `make_own_db.py`. Для его работы, в свою очередь, требуется API-ключ от TMDB (как для скрипта `hello_api_TMDB.py`). Процесс выглядит так:
1.  Получите API-ключ на [The Movie Database (TMDB)](https://www.themoviedb.org/settings/api) (инструкция в документации к `hello_api_TMDB.py`).
2.  Запустите скрипт `make_own_db.py`. Он скачает данные о фильмах с TMDB и сохранит их в файл `MyFilmDB.json` в той же папке. *Этот процесс может занять 15-20 минут.*
3.  После успешного создания файла `MyFilmDB.json` вы можете запускать `search_in_db.py`.

## Скрипт `find_similar.py`

## Назначение
Этот скрипт анализирует вашу базу данных фильмов и находит фильмы, наиболее похожие на тот, который вы укажете. Система рекомендаций работает на основе сравнения различных параметров фильмов:
- Принадлежность к одной коллекции (самый весомый параметр)
- Один и тот же оригинальный язык
- Схожий бюджет
- Совпадение по жанрам

Скрипт выводит список из 8 наиболее подходящих рекомендаций.

## Как его запустить?
1.  Убедитесь, что у вас установлен Python 3.
2.  Сохраните скрипт `find_similar.py` и вспомогательный модуль `own_db_helpers.py` в одну папку.
3.  **Убедитесь, что у вас есть файл базы данных** (например, `MyFilmDB.json`).
4.  Откройте терминал (командную строку) в этой папке.
5.  Выполните команду:
    ```bash
    python find_similar.py
    ```

## Что он требует для запуска?
Для работы скрипта необходимы:
- **Файл базы данных** в формате JSON (создается скриптом `make_own_db.py`)
- **Точное название фильма** на английском языке (как оно указано в базе данных)

## Где взять необходимые для его запуска штуки?
1.  **База данных**: создается скриптом `make_own_db.py` (требуется API-ключ от TMDB)
2.  **Точные названия фильмов**: можно узнать с помощью скрипта `search_in_db.py`, который покажет все доступные в базе фильмы и их точные названия

## Скрипт `find_similar.py`

### Назначение
Скрипт служит для поиска и вывода списка фильмов, максимально похожих на указанный пользователем фильм из локальной базы данных. Рекомендации формируются на основе комплексного сравнения нескольких параметров фильмов.

### Что он делает
- Принимает на вход путь к базе данных фильмов и точное название фильма для поиска похожих.
- Находит в базе фильм с введённым названием.
- Сравнивает все остальные фильмы в базе с этим фильмом, анализируя параметры:
  - Принадлежность к одной коллекции (наиболее значимый фактор).
  - Совпадение оригинального языка фильма.
  - Схожесть по бюджету.
  - Совпадение жанров.
- Каждому фильму присваивается оценка (рейтинг сходства) на основе совокупности перечисленных параметров и их значимости.
- Выводит список из 8 фильмов с наивысшим рейтингом — то есть наиболее похожих.

### Как он это делает (в общих чертах)
Скрипт использует набор правил и весов для параметров, которые определяют, насколько похожи два фильма. Например, если фильмы принадлежат к одной коллекции, они получают высокий балл, совпадение языка и жанров добавляют к этому рейтингу, а близкий бюджет влияет на итоговую оценку. Все эти факторы суммируются в конечный рейтинг, на основании которого формируется итоговый список.

### Для чего нужны ключевые числа и параметры (L11-L16)?
Это — веса, которые задают важность каждого отдельного параметра сравнения для формирования общей оценки сходства. Они помогают настроить алгоритм так, чтобы он учитывал наиболее значимые критерии для пользователя.

### Что такое рейтинг в скрипте? (L18-L23)
Рейтинг — это итоговое числовое значение, оценивающее степень похожести двух фильмов. Скрипт высчитывает рейтинг для каждого фильма относительно выбранного и использует его для сортировки рекомендаций. Чем выше рейтинг, тем фильм считается более похожим.

### Зачем нужна строка на L24?
Данная строка отвечает за инициализацию структуры или предустановку базового значения, необходимого для аккумулирования результатов сравнения и дальнейшей логики ранжирования фильмов.
